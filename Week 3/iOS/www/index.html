  • Week 2 Research
  • Twitter
    Twitter was as expected, very simple to work with. No token is needed to get tweets, since Twitter is all about being open with information. I would like a user to be able to sign in with their Twitter account to link the two. That way they can share tweets and other data such as pictures.
  • Instagram
    The Instagram API was a lot easier to use than I had assumed. The only thing that was different from Twitter was having to use an Access Token at the beginning. I basically gave permission for my app to access my account on Instagram (just for now). The idea I have is to let the user post pictures of their purchases, or to look up pictures of differnt kinds of spirits.

  • Week 3 Research
  • Notifications
    Notification will be used throughout the application. If a user is low on a certain alcohol then the program will remind them every few days to stock up. A user can also tell the application they don't want to be reminded of this.'
  • Accelerometer
    Accelerometer is used to detect is a user is shaking the device. During the shaking process the code will detect it and put a random recipe on a page. Hence the "Shake To Stir" recipe page.
  • Geo Location
    Geo Location is used to search around the users location for the closest stores that sell Alcohol.
  • Camera
    The Camera resource will be used in two instances. First it will be used as a barcode scanner so a user may scan a bottle of their favorite booze and automatically import the information. Second it will be used to take pictures of the booze you want to store for a virtual bar.
